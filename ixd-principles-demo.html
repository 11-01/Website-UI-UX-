👁️ 原则一：可见性（Visibility）
👆 按钮初始不显眼，hover才变色 — 违反可见性原则

✅ 正确做法

重要功能使用品牌色、足够大的尺寸、明确的视觉层级。
用户不需要"寻找"或"猜测"。

📢 原则二：反馈（Feedback）
0

每次操作都有即时反馈：变色/文案变化/数字弹跳

❌ 无反馈

用户困惑："到底点上了没有？"

🚫 原则三：限制（Constraints）

✅ 设计防错

限制输入范围 → 用户不可能输入无效值
按钮禁用 → 防止提交无效数据
"设计上让你不可能犯错"

🗺️ 原则四：映射（Mapping）
🔈 音量 ← 小   大 →
映射结果
🔉

🎯 滑块向右 → 音量变大 → 图标变大。逻辑和空间映射一致。

🔄 原则五：一致性（Consistency）

同一套按钮体系 → hover行为一致 → 用户一次学会全程通用

❌ 不一致

这个页面删除是红色，另一个页面删除是灰色
→ 用户肌肉记忆被打乱，每次都要重新判断

↩️ 原则六：容错性（Error Tolerance）

✅ 允许撤销

删除后出现撤销条 → 5秒内可恢复
给用户"后悔药"，降低操作焦虑

📋 六大原则速查
1. 可见性 重要功能显而易见，不要让用户猜测
2. 反馈 每次操作必须响应（变色/动画/提示）
3. 限制 通过设计防止用户犯错（禁用/格式限制）
4. 映射 控件与结果空间和逻辑保持一致
5. 一致性 相同元素相同行为，降低学习成本
6. 容错性 允许撤销，错误时提供友好恢复路径
