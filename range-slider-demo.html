核心 JS 逻辑

const slider = document.getElementById('mySlider');
const output = document.getElementById('sliderValue');

slider.addEventListener('input', () => {

});

Advanced Effects

📊 JS 进度填充效果

70

原生无"已滑动部分"CSS选择器，需通过 JS 动态设置 background: linear-gradient 来模拟进度条填充。

// JS 计算百分比更新背景渐变
const pct = (value - min) / (max - min) * 100;
slider.style.background = `linear-gradient(to right, #3b82f6 ${pct}%, #d1d5db ${pct}%)`;

开发避坑 + 移动端适配

Pitfalls & Mobile

📱 移动端 touch-action

iOS和部分安卓WebView中滑块可能触发页面滚动。建议CSS添加：

input[type="range"] {
  touch-action: pan-x;
}

并确保父容器没有 overflow: hidden 或复杂 transform 层级。

🎯 input vs change 事件

事件触发时机适用场景
input拖动中实时实时预览、数值显示
change松手后提交表单、API请求

⚠️ 进度填充的局限

原生无"已填充轨道"的CSS伪类选择器。所有进度条效果都必须通过 JS 动态计算百分比来实现。这是 <input type="range"> 最大的设计缺陷。

🧩 框架方案推荐

Element Plus: <el-slider v-model="val">
Ant Design: <Slider range defaultValue={[20,50]}>

框架组件已解决双滑块、刻度标记、输入框联动等复杂需求。
