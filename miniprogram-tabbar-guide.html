📏 数量 · 视觉 · 交互规范

🔢 数量限制

底部 Tab 2~5个 推荐≤4
顶部 Tab 5~7个 分类浏览
⚠️ 一个页面不应出现一组以上的标签分页栏

🎨 视觉规范

  • 图标尺寸：81×81px (@3x) 本地PNG
  • 图标风格：线性简洁，避免复杂渐变
  • 文字长度：≤4个汉字
  • 色彩对比：未选中(灰阶) vs 选中(主色填充)
  • 顶部Tab：仅文本，用下划线或字重区分选中态

🖱️ 交互规范

// ✅ 正确：Tab之间切换用 switchTab
wx.switchTab({ url: '/pages/index/index' });

// ❌ 错误：不能用 navigateTo / redirectTo
wx.navigateTo({ url: '/pages/index/index' });

// ⚠️ 切换触发 onShow，不是 onLoad
Page({
  onShow() { // 数据刷新写这里
    this.fetchUnreadCount();
  }
});

// 🔴 动态角标
wx.setTabBarBadge({ index: 2, text: '3' });
wx.showTabBarRedDot({ index: 1 });

📤 跨Tab传参

switchTab 不支持URL传参，替代方案： 

// 方案A：全局变量
const app = getApp();
app.globalData.selectedShop = shopId;

// 方案B：本地存储
wx.setStorageSync('targetShop', shopId);

// 方案C：事件总线
emitter.emit('shop:selected', shopId);
♿ 无障碍：为每个Tab配置 accessibility: { label: "首页" }，支持读屏软件朗读

二、自定义 Tab Bar 实现

自定义 何时使用自定义 Tab Bar？

✅ 适用场景
  • 异形/凸起中间按钮
  • 红点/数字角标提示
  • 渐变背景 / 毛玻璃效果
  • 动态切换Tab（如登录前后）
  • 特殊动效（点击粒子/水波纹）
❌ 不推荐场景
  • 常规4-5个tab的简单导航
  • 对性能有极致要求
  • 开发周期紧张、快速上线

📝 自定义 Tab Bar 组件代码

// ===== components/custom-tab-bar/index.wxml =====
<view class="tab-bar">
  <view wx:for="{{list}}" wx:key="index"
        class="tab-item {{item.selected ? 'active' : ''}}"
        bindtap="switchTab" data-index="{{index}}">
    <image src="{{item.selected ? item.selectedIconPath : item.iconPath}}"
           class="tab-icon" />
    <text class="tab-text">{{item.text}}</text>
    <!-- 红点角标 -->
    <view wx:if="{{item.badge}}" class="badge">{{item.badge}}</view>
  </view>
  <!-- 中间异形凸起按钮 --&span>
  <view class="center-btn" bindtap="onCenterTap">
    <image src="/static/icons/add.png" class="center-icon" />
  </view>
</view>

// ===== components/custom-tab-bar/index.wxss =====
.tab-bar {
  display: flex; align-items: flex-end;
  background: linear-gradient(180deg, #1e293b, #0f172a);
  padding-bottom: env(safe-area-inset-bottom);
}
.center-btn {
  width: 112rpx; height: 112rpx;
  background: linear-gradient(135deg, #f59e0b, #ef4444);
  border-radius: 50%; position: absolute; top: -56rpx;
  left: 50%; transform: translateX(-50%);
  box-shadow: 0 8rpx 32rpx rgba(239,68,68,.4);
}

// ===== components/custom-tab-bar/index.js =====
Component({
  data: {
    selected: 0,
    list: [
      { pagePath: "/pages/index/index", text: "首页",
        iconPath: "/static/icons/home.png",
        selectedIconPath: "/static/icons/home-active.png" },
      { pagePath: "/pages/map/map", text: "地图",
        iconPath: "/static/icons/map.png",
        selectedIconPath: "/static/icons/map-active.png" },
      { pagePath: "/pages/mine/mine", text: "我的",
        iconPath: "/static/icons/mine.png",
        selectedIconPath: "/static/icons/mine-active.png",
        badge: "3" }
    ]
  },
  methods: {
    switchTab(e) {
      const { index, item } = e.currentTarget.dataset;
      wx.switchTab({ url: item.pagePath });
    },
    onCenterTap() {
      wx.navigateTo({ url: '/pages/publish/publish' });
    }
  }
});

// ===== app.json 中注册（全局配置custom:true）=====
{
  "tabBar": {
    "custom": true,
    "list": [...]  // 仍需声明list用于编译，但渲染由自定义组件接管
  }
}

三、iOS / Android 边框一致性方案

📏 问题：Tab Bar 上边框在不同设备上粗细不一致

原因：iOS 和 Android 的 DPR（设备像素比）和渲染引擎不同，
导致 border-top: 1px 在高DPI屏幕上可能显示为2px甚至更粗。

✅ 三种解决方案对比

方案A：1rpx
直接使用 1rpx
在大部分设备上可达到物理1px
✅ 方案B：scaleY(0.5)
伪元素 + scaleY(0.5)
最精确，所有设备一致
方案C：0.5px
直接 0.5px
部分Android不识别(视为0)
/* ✅ 推荐写法：伪元素 + scaleY(0.5) 实现精确半像素边框 */
.tab-bar {
  position: relative;
}
.tab-bar::before {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0;
  height: 1px;                        /* 先用1px */
  background: rgba(0, 0, 0, 0.1);
  transform: scaleY(0.5);             /* 再缩放50%，变成精确0.5物理像素 */
  transform-origin: 0 0;
}

/* 备选方案（简单场景可用） */
.tab-bar {
  border-top: 1rpx solid rgba(0, 0, 0, 0.1);  /* rpx自动适配 */
}

🎯 选型决策速查表

场景 推荐方案 理由
常规企业应用 原生底部 性能最优，符合用户习惯
内容分类筛选 原生顶部 轻量文本切换，开发简单
品牌调性强烈 自定义TabBar 自由控制渐变/异形/动效
中间发布按钮 自定义TabBar 凸起异形按钮只能自定义实现
红点/角标通知 自定义TabBar 原生不支持动态角标
沉浸式首页 navigationStyle: custom 隐藏导航栏，全屏设计
