网站UI标签页设计完全指南

标签页（Tabs）是收纳和展现大块内容的经典组件。核心作用是"分类"与"切换"，保持界面整洁。

使用场景设计规范创意玩法 前端实现6种风格
1

明确使用场景 — 何时用/不用标签页

When to Use
场景说明
✅ 适用同层级不同类别内容如动物图集按类别展示，方便快速定位
✅ 适用信息过载需精简单次展示信息量过大时，用标签页分组降低认知负担
❌ 不适用内容无法明显分类标签间内容雷同，用户会误解分类含义
❌ 不适用需要对比内容用户需同时比较多块内容（如商品尺寸对比），应平铺展示
❌ 不适用有固定顺序流程如"选时间→选座位→支付"，应使用步骤条而非标签页

前端代码实现基础

Implementation

① HTML 结构

<!-- 标签按钮容器 -->
<div class="tab-nav">
  <button class="tab-btn active" data-tab="tab1">标签1</button>
  <button class="tab-btn" data-tab="tab2">标签2</button>
</div>
<!-- 内容面板 -->
<div class="tab-panel active" data-panel="tab1">...</div>
<div class="tab-panel" data-panel="tab2">...</div>

② CSS 核心

/* 默认隐藏面板 */
.tab-panel { display: none; }
.tab-panel.active { display: block; }

/* 切换动画 */
.tab-panel.active {
  animation: fadeIn .3s ease;
}
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(6px); }
}

③ JavaScript 逻辑

// 监听标签点击事件
document.querySelectorAll('.tab-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    const tabGroup = btn.closest('.tabs-underline');
    // 1. 移除所有按钮和面板的 active 类
    tabGroup.querySelectorAll('.tab-btn, .tab-panel').forEach(el => el.classList.remove('active'));
    // 2. 激活当前按钮
    btn.classList.add('active');
    // 3. 显示对应面板
    tabGroup.querySelector(`[data-panel="${btn.dataset.tab}"]`).classList.add('active');
  });
});
💡 企业级项目建议：直接使用成熟的 UI 组件库（Ant Design · Arco Design · Bootstrap），它们已封装响应式适配、无障碍访问（A11y）和复杂交互动画。
