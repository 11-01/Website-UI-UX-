视觉设计原则 — 统一克制，主次分明

Design Principles

🎨 色彩与排版

严格贴合企业VI，全站主色调≤4种。正文用深灰(#333)而非纯黑，提升阅读柔和度。字体统一，层级清晰。

📐 留白与网格

采用网格标准化布局，所有元素对齐规整。留白适度 — 既不拥挤堆砌，也不过度空旷，保持页面"呼吸感"。

📌 吸顶导航 (Sticky)

页头固定顶部，滚动时可调整背景透明度或简化高度，确保随时可切换页面且不遮挡核心内容。

✨ 轻量化动效

动效应为体验加分而非干扰。仅设置基础hover悬浮、平滑滚动渐入等微动效，杜绝闪烁或夸张弹跳。

吸顶导航与响应式适配

Sticky & Responsive
Sticky Header 实时演示 — 滚动下方区域观察吸顶效果
🏢 Brand 首页产品关于 登录

📜 向下滚动观察 — 页头始终固定在顶部

使用 position: sticky; top: 0; 实现吸顶效果。

当用户向下滚动时，导航栏固定在顶部，确保随时可以切换页面。

可配合 JS 监听滚动位置，动态调整导航栏的背景透明度或高度。

例如：滚动超过 100px 后，减小 Logo 尺寸，压缩导航栏高度。

--- 内容分隔线 ---

在移动端（屏幕宽度 < 768px），主导航应折叠为汉堡菜单。

确保所有可点击元素的触控区域 ≥ 44×44px，防止误触。

--- EOF ---

核心 CSS
.header {
  position: sticky;
  top: 0;
  z-index: 100;
  background: rgba(255,255,255,.95);
  backdrop-filter: blur(10px);
}
✅ 最佳实践：sticky 优于 fixed — 不脱离文档流，无需额外设置 margin/padding 补偿。配合 backdrop-filter: blur() 实现毛玻璃效果更佳。
5

技术实现思路（HTML+CSS）

Implementation

① HTML 语义化结构

<header>
  <nav>
    <ul>
      <li><a href="#">首页</a></li>
    </ul>
  </nav>
</header>

<ul> > <li> > <a> 比直接用 <a> 更语义化，利于 SEO。

② CSS 布局

.header-inner {
  max-width: 1200px;
  margin: 0 auto;
  display: flex;
  align-items: center;
  justify-content: space-between;
}

版心宽度（如1200px）水平居中，Flexbox 弹性布局水平排列。

③ 导航链接间距

.nav a {
  padding: .5rem 1rem;
  /* 不设 width，用 padding 撑开 */
}

不直接指定宽度，通过左右 padding 撑开盒子，方便增减文字。

④ 响应式汉堡菜单

@media (max-width: 768px) {
  .nav-links { display: none; }
  .hamburger { display: block; }
}

移动端隐藏导航链接，显示汉堡图标，点击展开。
