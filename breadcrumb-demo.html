✅ 正确写法

<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">首页</a></li>
    <li><a href="/blog">博客</a></li>
    <!-- 当前页：span + aria-current -->
    <li><span aria-current="page">文章标题</span></li>
  </ol>
</nav>
/* 分隔符用CSS ::after */
li + li::before { content: "›"; }

❌ 错误写法

<!-- 错误1：div堆砌，无语义 -->
<div class="breadcrumb">
  <a href="/">首页</a> >
  <a href="/blog">博客</a> >
  <a href="/blog/post">文章</a>   ← 当前页竟可点击
</div>
<!-- 错误2：分隔符 > 直接写在HTML中 -->
<!-- 屏幕阅读器会朗读"大于号" -->
<!-- 错误3：无 aria-current，无结构化数据 -->

🔍 SEO：Schema.org 结构化数据 (JSON-LD)

将以下JSON-LD放在页面 <head> 中，搜索引擎会在搜索结果中展示面包屑路径，显著提升点击率 

{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "首页",
      "item": "https://example.com/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "智能手机",
      "item": "https://example.com/phones/"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "iPhone 16 Pro"
    }
  ]
}
🔍 搜索结果会显示：example.com › 智能手机 › iPhone 16 Pro
🌲 面包屑 vs 目录树 — 不是一回事
对比维度 🍞 面包屑 🌲 目录树
形态 水平线性排列 垂直树状结构
展示 仅一条路径（根→当前） 全部节点，支持多分支
功能 定位当前位置 + 快速返回 全局浏览 + 切换分支
交互 点击链接跳转 展开/折叠节点
空间 极简一行，几乎不占空间 占据侧栏等较大垂直空间
📋 面包屑设计决策速查
网站层级 ≥ 3？→ 需要面包屑
电商/内容站？→ 位置型 + 属性型组合
向导/多步表单？→ 步骤型面包屑
有SEO需求？→ 必须加JSON-LD结构化数据
最后一项？→ <span aria-current="page">，不可点击
